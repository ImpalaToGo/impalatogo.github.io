A standalone distribution of Impala, optimized to work with cloud storage, bypassing HDFS completely.

How To Try

ImpalaToGo

Fork of Cloudera Impala, separated from hadoop. It is optimized to work in cloud with S3 storage by caching data locally on cluster nodes.

Why ImpalaToGo

  1. It is Impala without hadoop. You can take advantage of its fast query engine without managing whole hadoop stack.
  2. Optimized to work with S3. ImpalaToGo transparently cache data on local drives of cluster nodes.
  3. Optimized to work in cloud. ImpalaToGo has simple interface to start cluster in cloud.
  4. Actually the only open source MPP database written in C++
  5. It gives you almost the same capabilities as Hive over S3, but much faster.

How to try: https://github.com/ImpalaToGo/ImpalaToGo/wiki/How-to-try

Short presentation on ImpalaToGo architecture: http://www.slideshare.net/DavidGroozman/impala-togo-introduction

What we add to Cloudera Impala?

We are developing caching layer which use local drives to cache access to remote storage. Its advantage can be easy to be seen when working with S3.
